## 2. データ共有アーキテクチャ

### 2-1. 共有ファイルと参照方法

| ファイル | WEB版の参照パス | Mobile版の参照パス |
|----------|----------------|-------------------|
| `japan_history_data.json` | `'japan_history_data.json'`（同階層） | `'../japan_history_data.json'`（一つ上） |
| `japan.topojson` | `'japan.topojson'`（CDNフォールバックあり） | `'../japan.topojson'`（フォールバックなし） |

**両バージョンが同一ファイルを読んでいるため、JSONをコミットすれば両方に反映される。**

### 2-2. WEB版のデータ読込フロー（3段階優先順位）

```
① インライン最小 seed（index.html内のハードコード — 非常時フォールバック）
       ↓ 上書き
② fetch('japan_history_data.json', {cache:'no-store'})
   ← サーバー最新を毎回取得（JSONが正のマスター）
       ↓ マージ（version比較）
③ localStorage（ユーザーの未保存編集）
   ← JSON version > localStorage version の場合は localStorage を破棄し JSON 優先
```

- `saveData()` : 編集のたびに localStorage に保存
- `exportData()` : `version+1` で JSON ファイルとしてダウンロード → リポジトリに差し替えてコミット
- `importData()` : JSONファイルを読み込んで全データ上書き

### 2-3. Mobile版のデータ読込フロー

```
fetch('../japan_history_data.json') + fetch('../japan.topojson')
  ↕ ネットワークファースト（オンライン時は常に最新取得）
Service Worker キャッシュ（オフライン時フォールバック）
```

- localStorage は使用しない
- 編集機能なし（閲覧専用）

### 2-4. データ更新の正しいフロー

```
WEB版で編集
  → 「JSONエクスポート」でダウンロード（version が自動インクリメント）
  → リポジトリの japan_history_data.json に差し替え
  → git commit & push
  → 両バージョンに自動反映
      WEB版：次回アクセス時に fetch で最新取得（localStorage より version が大きければ優先）
      Mobile版：次回アクセス時に SW ネットワークファーストで最新取得
```

⚠️ **注意**: JSON を直接手編集した場合（Pythonスクリプト等）は `version` フィールドを手動でインクリメントすること。インクリメントしないと WEB 版の localStorage 内旧データが優先されるユーザーが発生する。

## 3. JSON スキーマ詳細（japan_history_data.json）

### 3-1. トップレベル

```json
{
  "version": 2,
  "persons": [...],
  "emperors": [...],
  "rulers": [...],
  "cats": [...]
}
```

| フィールド | 型 | 説明 |
|-----------|---|------|
| `version` | number（整数） | データバージョン。エクスポート時に自動 +1。WEB版の localStorage 上書き判定に使用 |
| `persons` | array | 人物（文化人・僧侶等） |
| `emperors` | array | 天皇・上皇 |
| `rulers` | array | 為政者（将軍・政治家等） |
| `cats` | array | カテゴリ（宗教・芸能・出来事等）とその配下イベント |

### 3-2. persons の各オブジェクト

```json
{
  "id": "zeami",
  "n": "世阿弥",
  "fg": "ぜあみ",
  "l": "1363〜1443",
  "era": "muromachi",
  "role": "能楽師・劇作家",
  "desc": "能楽を大成した...",
  "locs": [...]
}
```

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `id` | ✅ | 一意な英数字キー（URL safe） |
| `n` | ✅ | 表示名（漢字） |
| `fg` | — | ふりがな（検索・ドロップダウンに使用） |
| `l` | — | 生没年テキスト（表示用のみ） |
| `era` | — | 時代ID（後述の Era ID 一覧参照） |
| `role` | — | 肩書・役職 |
| `desc` | — | 説明文（200字以内推奨） |
| `locs` | — | 地図マーカー情報（後述） |

### 3-3. emperors / rulers の各オブジェクト

```json
{
  "id": "ojin",
  "n": "応神天皇",
  "l": "270〜310年頃",
  "s": 270,
  "e": 310,
  "role": "第15代天皇",
  "desc": "...",
  "locs": [...]
}
```

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `id` | ✅ | 一意な英数字キー |
| `n` | ✅ | 表示名 |
| `l` | — | 期間テキスト（表示用） |
| `s` | — | 開始年（西暦、マイナス可） |
| `e` | — | 終了年（西暦） |
| `role` | — | 肩書 |
| `desc` | — | 説明文 |
| `locs` | — | 地図マーカー情報 |

※ `era` フィールドは emperors/rulers には存在しない。`s`/`e` で直接年代指定。

### 3-4. locs（位置情報）オブジェクト

```json
{
  "r": "yamashiro",
  "tp": "active",
  "c": "#6088ff",
  "lb": "京都にて能を大成"
}
```

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `r` | ✅ | 旧国名コード または 地域コード（後述） |
| `tp` | ✅ | `"birth"` / `"active"` / `"exile"` / `"death"` |
| `c` | ✅ | カラーコード（WEB版詳細画面の色。Mobile版は tp から自動決定） |
| `lb` | ✅ | 表示ラベル（テキスト。両版で使用） |

#### `tp` の対応

| tp 値 | 意味 | WEB版 `c` 推奨色 | Mobile版マーカー色 |
|-------|------|-----------------|-------------------|
| `birth` | 出生地 | `#50cc50` | `#40b060`（緑） |
| `active` | 活躍の地 | `#6088ff` | `#5090ff`（青） |
| `exile` | 流刑地 | `#ff8050` | `#ff8030`（オレンジ） |
| `death` | 没地 | `#909090` | `#909090`（グレー） |

⚠️ Mobile版は `c` フィールドを**無視**して `tp` から色を決定する。`c` はWEB版用。

#### `r` フィールドの値体系（★重要）

`r` は以下の2種類が混在する。**精度の観点から旧国名コードを優先して使うこと。**

**① 旧国名コード**（推奨・精度高）
→ Mobile版では `PROVPREF[r]` → 都道府県名 → SVGセントロイドにマッピング

| コード例 | 旧国名 | マップ先都道府県 |
|---------|--------|----------------|
| `yamashiro` | 山城国 | 京都府 |
| `yamato` | 大和国 | 奈良県 |
| `musashi` | 武蔵国 | 東京都・埼玉県・神奈川県 |
| `sagami` | 相模国 | 神奈川県 |
| `sado` | 佐渡国 | ※特別処理（後述） |
| `chikuzen` | 筑前国 | 福岡県 |
| `iga` | 伊賀国 | 三重県 |
| `settsu` | 摂津国 | 大阪府・兵庫県 |
| `awa_k` | 安房国 | 千葉県 |
| `echizen` | 越前国 | 福井県 |
| `sanuki` | 讃岐国 | 香川県 |
| `satsuma` | 薩摩国 | 鹿児島県 |
| `kii` | 紀伊国 | 和歌山県・三重県 |
（全リストは `mobile/index.html` の `PROVPREF` 定数参照。約70国）

**② 地域コード**（精度低・広域概念）
→ Mobile版では `rgCentroids_[r]`（地域内都道府県の平均セントロイド）

| コード | 地域 | 該当都道府県 |
|--------|------|------------|
| `kanto` | 関東 | 茨城〜神奈川 |
| `kinki` | 近畿 | 三重・滋賀・京都・大阪・兵庫・奈良・和歌山 |
| `chubu` | 中部・北陸 | 新潟〜静岡・愛知 |
| `tohoku` | 東北 | 青森〜福島 |
| `chugoku` | 中国 | 鳥取〜山口 |
| `kyushu` | 九州 | 福岡〜沖縄 |
| `shikoku` | 四国 | 徳島〜高知 |
| `hokkaido` | 北海道 | 北海道 |

⚠️ 地域コードを使うと、同地域内の複数人物が**同じ座標**に重なる。できる限り旧国名コードを使うこと。  
⚠️ emperors/rulers は現状まだ多くが地域コードのまま（今後精緻化余地あり）。

**③ 離島の特別処理（KUNI_LNGLAT）**

| `iki` | 壱岐 | 129.72°E, 33.74°N |

### 6-2. 地図マーカー表示ロジック

```javascript
// drawPersonLocsOnMap() の処理順序
1. KUNI_LNGLAT[r] があれば projection([lng,lat]) でSVG変換（離島優先）
2. PROVPREF[r] があれば prefCentroids[pref] で都道府県セントロイド
```

### 6-3. Service Worker（SW）

- ファイル: `mobile/sw.js`

## 7. WEB版 固有の仕様

### 7-1. データ管理

- **localStorage**: キー `jhm_data`。ユーザー編集を自動保存。
- **エクスポート**: `exportData()` → `version+1` の JSON をダウンロード。
- **インポート**: `importData(input)` → ファイルから全データ上書き。
- **JSON優先ルール**: `JSON.version > localStorage.version` のとき localStorage を破棄。

### 7-2. 編集機能

- イベント追加・編集・削除
- カテゴリ追加・編集・削除
- 人物（persons/emperors/rulers）追加・編集・削除
- 編集後は `saveData()` で localStorage に自動保存

### 7-3. 地図（東日本・西日本）

WEB版の地図は Mobile版とは別実装。TopoJSON は同一ファイルを参照。
WEB版の地図マーカーは `lb` のテキスト表示のみ（`r` フィールドは位置決定に使わない）。

## 8. データ追加・修正時のチェックリスト

### persons/emperors/rulers を追加する場合

- [ ] `id` がリスト内で一意か確認
- [ ] `n`（表示名）を設定
- [ ] `locs` の `r` は**旧国名コード**を使う（地域コードは精度が落ちる）
- [ ] 離島（佐渡・対馬・壱岐・隠岐・淡路）は専用コードが KUNI_LNGLAT に登録済み
- [ ] 同一人物で同じ旧国名コードを複数 locs に使う場合 → Mobile版で自動的に交互フェード表示になる（正常動作）
- [ ] `tp` は `birth` / `active` / `exile` / `death` のいずれか
- [ ] `c` は WEB版用カラー（Mobile版は `tp` から自動決定するので参考値でよい）

### イベントを追加する場合

- [ ] `id` が cats 内で一意か確認
- [ ] `s`/`e` は西暦整数
- [ ] `c` はバー色（`#RRGGBB` 形式）

### JSONをコミットする場合

- [ ] `version` フィールドをインクリメント（WEB版 localStorage 上書きのため必須）
- [ ] `mobile/sw.js` の `CACHE = 'jhm-vN'` を N+1 に上げる（Mobile版キャッシュクリア）
- [ ] `japan_history_data.json` は minified（改行なし）で保存（Python `json.dump` 使用時: `separators=(',',':')` を指定）
